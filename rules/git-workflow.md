# git workflow rules

## identity (CRITICAL — never violate)
- define your git identities per context (personal vs work org)
- example: personal repos use `yourname` / `yourname@users.noreply.github.com`
- example: work repos use `yourname-org` as GitHub identity
- never auto-switch identity. always verify before first commit in a session.

## commit format
- use conventional commits: `type(scope): description`
- types: feat, fix, refactor, test, docs, chore, perf, ci
- description in lowercase, imperative mood, max 72 chars
- NEVER mention "Claude", "Claude Code", "AI", or "generated by" in any commit message
- NEVER mention "Claude", "Claude Code", "AI", or "generated by" in any PR title or body
- no emojis in commit messages

## pull requests
- PR title: lowercase, conventional commit format
- PR body: concise summary of what and why (not how)
- always reference ticket if one exists (e.g., `Closes PROJECT-1234`)
- keep PR body under 300 words unless architectural change
- never include "Co-authored-by" trailer referencing Claude or AI tools

## branching
- feature branches: `feat/TICKET-description` or `feat/description`
- fix branches: `fix/TICKET-description` or `fix/description`
- always rebase on main before PR
